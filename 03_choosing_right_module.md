# Choosing modules

- We have to choose the right set of modules, from the large set of modules
- The rules we need to keep in mind for choosing modules are


## 1. Check the number of downloads in terraform registry
- indicates lot of individulas and organizations are downloading and using the module


## 2. Check the github page. It should contain many contributors
- Well maintained modules will have good amount of contributors
- It will not have single contributor
- It should also be regularly updated, commits should be recent

## 3. Check the github page. It should have good amount of fork and stars.
- This will indicate community intrest and builds confidence to use the module


## 4. Check the github page for issues, It should have good amount of open issues and closed/fixed issues
- indicates the consistent updates to the repository and bug fixes


## 5. Check the source code of the module
- The source code shoudld be professionally written

## 6. Check the github readme
- well maintained module will have extensive readme on how to use the module, examples and all the information in detaileed format.

## 7. In terraform registry, we can check the version history of the module, 
- The module should have good number of version history
- indicates constant update to the module and maintanance

## 8. Use modules maintained by Hashicorp partners
- It will have hashicorp partner badging
- It will be trustworthy modules and we can use it

# Important point to note

## Never download any random module which has single contributor and no downloads as it can be an attacker
## They can send the account information to the attacker when the module is executed.

# Which modules organization use? 

- Organizations maintan thier own set of modules internally
- They will fork from a well maintained public module and maintain it internally
